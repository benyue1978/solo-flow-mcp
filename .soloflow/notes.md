## 🧩 架构设计笔记

### 设计原则
1. **模块化**: 清晰的职责分离
2. **可扩展**: 支持新工具和提示的轻松添加
3. **安全性**: 严格的数据验证和访问控制
4. **性能**: 轻量级实现，快速响应
5. **标准化**: 遵循 MCP 协议标准

### 关键技术决策
- **选择 TypeScript**: 提供类型安全和更好的开发体验
- **使用 Jest**: 成熟的测试框架，支持异步测试
- **StdioServerTransport**: 适合本地 IDE 集成
- **文件系统存储**: 简单可靠，无需外部依赖

### 架构优势
- **本地优先**: 无需网络连接，完全本地运行
- **轻量级**: 最小化依赖，快速启动
- **跨平台**: 支持 Windows, macOS, Linux
- **易集成**: 与 Cursor 等 IDE 无缝集成
